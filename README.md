# Employee Performance Mapping — MySQL

## 📌 Objective
Analyze employee performance ratings, salary 
structures, and reporting hierarchies to support 
HR decision-making using advanced SQL techniques.

## 🛠️ Tools Used
- MySQL | SQL Workbench
- Window Functions | Views | Joins

## 📊 Key Analysis Performed
- Filtered employees by performance ratings
  (< 2, between 2-4, > 4) for HR review
- Identified managers and direct reportees
  using Self JOIN technique
- Ranked employees by experience using
  RANK() and DENSE_RANK() Window Functions
- Analyzed MIN/MAX salary by role using
  PARTITION BY clause
- Created department-wise performance views
  for leadership reporting
- Combined Healthcare & Finance data
  using UNION operations

## 🔑 SQL Concepts Demonstrated

| Concept | Usage |
|---|---|
| Self JOIN | Manager-employee hierarchy mapping |
| Window Functions | RANK, DENSE_RANK, PARTITION BY |
| UNION | Cross-department employee listing |
| CREATE VIEW | Salary-filtered employee view |
| GROUP BY + Aggregations | Department-wise analysis |
| CONCAT | Full name string formatting |
| WHERE + Filtering | Rating-based segmentation |

## 💡 Business Insights Generated
- Mapped complete reporting hierarchy 
  from CEO to junior employees
- Highlighted salary ranges across roles
- Identified performance distribution 
  across all departments
- Created reusable views for HR reports
